> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix clustering false merges and null `why_it_matters` by expanding entity coverage from 6% to 70%+, tightening MLT matching, and stopping the CVE snowball. All improvements are heuristic — no external AI APIs required.

**Architecture:** Three tasks, all pure heuristics. Task 1 pulls threat actor/malware/tool data from MITRE ATT&CK (downloaded once, stored locally as `data/threat_keywords.json`) so the gazetteer stays current without code changes. Task 2 adds stop words and a size cap to MLT. Task 3 anchors CVE matching to a cluster's founding article CVEs only. The system works with or without `data/threat_keywords.json` (falls back to a small hardcoded baseline).

**Tech Stack:** Python 3.12, opensearch-py (async), requests (stdlib-compatible download), pytest + pytest-asyncio

## Task 1: MITRE ATT&CK-Driven Gazetteer + Alias Normalization

**Context:** `entity_extractor.py` has 35 hardcoded threat actors/malware/tools. Most 2024-2026 articles mention actors and malware not in the list, and aliases like "Midnight Blizzard" (APT29) and "Forest Blizzard" (APT28) never match the canonical keys — so two articles about the same actor never entity-overlap cluster. MITRE ATT&CK already has hundreds of groups/malware/tools with all their aliases pre-mapped in a free, downloadable STIX JSON. This task adds a sync script that generates `data/threat_keywords.json` from MITRE data and modifies `entity_extractor.py` to load from it at startup, falling back to a minimal hardcoded baseline if the file is absent.

```python
#!/usr/bin/env python
"""Download MITRE ATT&CK STIX and generate data/threat_keywords.json.

Downloads the enterprise-attack STIX bundle from the mitre/cti GitHub repo,
extracts groups (actors), malware, and tools with their aliases, and writes
data/threat_keywords.json for use by entity_extractor.py.

Usage:
    python scripts/sync_mitre_attack.py
    python scripts/sync_mitre_attack.py --output path/to/custom.json
"""
import argparse
import json
import logging
import re
import sys
from pathlib import Path

import requests

logger = logging.getLogger(__name__)

STIX_URL = (
    "https://raw.githubusercontent.com/mitre/cti/master/"
    "enterprise-attack/enterprise-attack.json"
)

_DEFAULT_OUTPUT = Path(__file__).resolve().parent.parent / "data" / "threat_keywords.json"


def _normalize(name: str) -> str:
    key = name.lower()
    key = re.sub(r"[^a-z0-9]+", "-", key)
    return key.strip("-")


def _build_threat_data(stix: dict) -> dict:
    keywords: dict[str, list] = {}
    aliases: dict[str, str] = {}

    for obj in stix.get("objects", []):
        if obj.get("revoked") or obj.get("x_mitre_deprecated"):
            continue

        obj_type = obj.get("type")
        name = obj.get("name", "").strip()
        if not name:
            continue

        if obj_type == "intrusion-set":
            etype = "actor"
            obj_aliases = obj.get("aliases") or []
        elif obj_type == "malware":
            etype = "malware"
            obj_aliases = obj.get("x_mitre_aliases") or []
        elif obj_type == "tool":
            etype = "tool"
            obj_aliases = obj.get("x_mitre_aliases") or []
        else:
            continue

        canonical_key = _normalize(name)
        keywords[canonical_key] = [name, etype]

        for alias in obj_aliases:
            alias = alias.strip()
            if alias and alias != name:
                aliases[alias] = canonical_key

    return {"keywords": keywords, "aliases": aliases}


def main(output_path: Path) -> None:
    logger.info("Downloading MITRE ATT&CK STIX from GitHub...")
    resp = requests.get(STIX_URL, timeout=60)
    resp.raise_for_status()
    stix = resp.json()
    logger.info("Downloaded %d STIX objects.", len(stix.get("objects", [])))

    data = _build_threat_data(stix)
    logger.info(
        "Parsed %d keywords, %d aliases.",
        len(data["keywords"]),
        len(data["aliases"]),
    )

    output_path.parent.mkdir(parents=True, exist_ok=True)
    with open(output_path, "w") as f:
        json.dump(data, f, indent=2, sort_keys=True)
    logger.info("Written to %s", output_path)


if __name__ == "__main__":
    logging.basicConfig(level=logging.INFO, format="%(levelname)s %(message)s")
    parser = argparse.ArgumentParser()
    parser.add_argument("--output", type=Path, default=_DEFAULT_OUTPUT)
    args = parser.parse_args()
    main(args.output)
```

```python
    # Report boilerplate
    "advisory", "bulletin", "alert", "roundup", "brief", "summary", "report",

    "disclosed", "disclosure", "researchers", "researcher", "cve", "cvss",
    "mitigation", "mitigate", "mitigated", "detection", "protect", "protection",
]

_MLT_MAX_CLUSTER_SIZE = 15  # clusters larger than this cannot absorb via MLT


async def find_cluster_by_mlt(
    title: str, summary: Optional[str], window_hours: int = 24,
) -> Optional[str]:
    """Find a narratively similar cluster via more_like_this."""
    like_text = title
    if summary:
        like_text = f"{title} {summary}"

    client = get_os_client()

    # Guard: MLT needs a minimum corpus to produce meaningful results
    count_resp = await client.count(index=INDEX_CLUSTERS)
    if count_resp.get("count", 0) < 20:
        return None

    cutoff = (datetime.now(timezone.utc) - timedelta(hours=window_hours)).isoformat()

    resp = await client.search(
        index=INDEX_CLUSTERS,
        body={
            "query": {
                "bool": {
                    "must": [
                        {
                            "more_like_this": {
                                "fields": ["label", "summary"],
                                "like": like_text,
                                "min_term_freq": 1,
                                "min_doc_freq": 1,
                                "minimum_should_match": "30%",
                                "stop_words": _MLT_STOP_WORDS,
                            }
                        }
                    ],
                    "filter": [
                        {"range": {"created_at": {"gte": cutoff}}},
                        {"range": {"article_count": {"lte": _MLT_MAX_CLUSTER_SIZE}}},
                    ],
                }
            },
            "size": 1,
            "_source": False,
        },
    )

    hits = resp["hits"]["hits"]
    if hits and hits[0]["_score"] >= _MLT_SCORE_THRESHOLD:
        return hits[0]["_id"]
```

## Task 3: Seed-Based CVE Gating

**Context:** Currently, when Article B merges into a cluster, its CVEs are appended to `cluster.cve_ids`. Future articles matching any of those accumulated CVEs get pulled in via Tier 1 CVE check — even if they cover a completely different story. This task adds `seed_cve_ids` to the cluster schema: only the founding article's CVEs are used for matching. The accumulated `cve_ids` pool remains for display/search but is never used for cluster lookup.

## Future: AI Integration (deferred)

When a suitable local model is available (not a cloud API), two AI modules can be added. Both must be **fully optional** — the system must work identically when neither is present.

### AI NER fallback (`app/ingestion/ai_extractor.py`)

Called from `ingester.py` only when regex extraction finds zero entities. Must return the same `list[dict]` format as `extract_entities()` (keys: `type`, `name`, `normalized_key`). Gated by an env var (e.g. `LOCAL_NER_ENDPOINT`). When not configured, returns `[]` silently.

### AI cluster summarization (`app/ingestion/ai_summarizer.py`)

Called from `merge_into_cluster` after `rescore_cluster` when a cluster transitions to confirmed state (≥3 articles) and `why_it_matters` is still null. Must return `Optional[dict]` with keys `"why_it_matters"` and `"summary"`, or `None` on failure. Gated by the same env var. Failure must never raise — wrap in try/except and log a warning.

Both modules must have their own test files with mocked model calls.
